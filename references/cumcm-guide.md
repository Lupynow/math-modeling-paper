# 国赛 (CUMCM) 详细写作指南

## 评审机制与写作策略

国赛评审流程：第一轮看摘要和整体结构筛选 → 第二轮细读正文 → 第三轮答辩（高奖候选）。

这意味着：
- **摘要决定生死**：摘要不好，正文写得再好评委看不到
- **结构完整性**：缺少模型检验、参考文献不规范是明确扣分点
- **量化结果**：没有具体数值的论文不可能拿高奖

## 各章节详细写法

### 一、摘要（1页，约1000字）

**结构模板**：

```
第一段（2-3句）：针对[题目核心问题]，本文首先[数据处理方法]，在此基础上分别建立[模型1]、[模型2]和[模型3]，对[问题概述]进行系统求解。

第二段：针对问题一，[方法]建立[模型]，得到[具体结果+数值]。结果表明，[关键发现]。

第三段：针对问题二，在问题一基础上，[方法]建立[模型]，求得[具体数值]。对比显示，[核心对比结论]。

第四段：针对问题三，进一步考虑[因素]，采用[方法]得到[具体结果]。

第五段（1-2句）：通过[检验方法]验证了模型的有效性。灵敏度分析表明[关键参数]对结果影响[程度]，模型具有[良好/一定]的稳定性。

关键词：[3-5个，如多目标优化、灰色预测、灵敏度分析、综合评价]
```

**强制检查清单**：
- [ ] 问题一的结果有具体数字吗？
- [ ] 问题二的结果有具体数字吗？
- [ ] 问题三的结果有具体数字吗？
- [ ] 有模型检验/灵敏度分析的提及吗？
- [ ] 关键词完全了吗？

### 二、问题重述（0.5-1页）

不要复制题目。写法：
1. 用1-2句话概括整体背景
2. 逐个子问题用自己的话重新表述，每个2-3句
3. 注意"重述"而非"缩写"——要体现出你对问题的理解

### 三、问题分析（1-2页）

按问题分开写。每个问题的分析包含：
- 这个问题的数学本质（预测？评价？优化？分类？机理？）
- 输入是什么、输出需要什么
- 为什么选这个模型（要对比至少一种替代方案）
- 本问题的难点在哪里
- 与前一个问题的关系（如果是问题二或三）

**示例写法（好）**：
> 问题一要求评价不同方案的综合效益。这是一个典型的多指标综合评价问题。评价类问题常用方法有层次分析法(AHP)、TOPSIS法和模糊综合评价法。考虑到本问题指标数量较多且部分指标具有一定模糊性，本文选用模糊综合评价法，既能处理定量指标也能处理专家对定性指标的模糊判断。指标权重的确定采用熵权法以降低主观性。

**示例写法（差）**：
> 针对问题一，我们首先进行数据处理，然后使用模糊综合评价法进行评价，最后得到评价结果。

### 四、模型假设与符号说明（1页）

**假设写作原则**：
1. 每个假设是可检验的——在模型检验部分可以验证是否成立
2. 每个假设服务于建模——去掉这个假设会导致模型复杂到不可解
3. 不写"假设数据真实可靠"这类不服务建模的废话

**符号说明三线表**，必须包含：符号 | 含义 | 单位

### 五、数据处理（2-4页）

如果题目给了数据，这一部分请写清楚：
1. 数据来源和初步观察
2. 缺失值：哪些变量有缺失，比例多少，用什么方法填补，为什么
3. 异常值：检测方法（3σ/IQR/箱线图），处理方式（删除/修正/保留），为什么
4. 标准化：方法选择（Z-score/Min-Max/其他），原因
5. 特征提取/降维（如有）

每种处理都要说明"为什么这样处理"。例如不清楚量纲差异有多大，就先用描述性统计展示量纲差异，再说明标准化是必要的。

### 六、模型建立与求解（每个问题4-7页）

每个子问题按以下顺序展开：

**6.1 问题目标**：本子问题要解决的核心问题是什么，一句话说明

**6.2 模型选择**：为什么选这个模型？需要对比至少一种替代方案。例如：
> 时间序列预测可选用ARIMA、LSTM或Prophet。ARIMA适合平稳序列且可解释性强但要求序列平稳；LSTM能捕捉非线性依赖但需要较大数据量且有黑箱问题；Prophet适合有季节性的商业预测但对短期波动捕捉不足。考虑到本题数据XX特点，选用XX。

**6.3 模型建立**：从题目出发推导公式。每个变量要对应到题目的具体概念。不是抄教科书的标准形式。

**6.4 参数说明**：每个参数的值和取值依据。

**6.5 求解**：求解方法、关键步骤（可画流程图）、收敛条件/停止准则。

**6.6 结果展示**：表格（三线表）+ 图表。表格要有清晰的表头和单位。图表要有描述性标题。

**6.7 结果分析**：这个结果说明了什么？是否符合预期？为什么？

### 七、模型检验（2-3页，必须写）

参考 `references/model-validation.md` 的完整方法指南。国赛至少包含：

1. **误差分析**（必需）：MSE/MAE/R²，或相对误差分析
2. **灵敏度分析**（强烈推荐）：关键参数变化对结果的影响
3. **对比验证**（推荐）：用简单方法/替代方法和你的模型对比

### 八、优缺点与改进（1页）

优点3-5条，缺点3-5条。每一条必须以"本模型……"开头，针对你的具体建模选择。

改进建议要有建设性：不是"可以用更好的方法"，而是"后续可将XX假设替换为XX分布，引入XX算法处理XX约束"。

### 九、参考文献

最少8条。组成建议：3-4篇中文期刊 + 2-3篇英文期刊 + 1-2本教材 + 1篇其他

### 十、附录

- 完整源代码（可以运行的，不要删减）
- 支撑材料文件列表
- 较长的中间结果表（如果正文放不下）

### 国赛排版要点

- 正文用宋体小四，标题用黑体
- 公式居中，编号右对齐
- 表格用三线表
- 图表标题：表标题在上方，图标题在下方
- 页边距符合规范（格式模板中的要求）
