# A/B/C 题差异化策略

国赛（CUMCM）每年设置 A/B/C 三道题，覆盖不同的数学建模领域。本题型策略基于 2018-2024 年 70+ 篇获奖论文的分析提炼。

---

## 题型概览

| 维度 | A 题 | B 题 | C 题 |
|------|------|------|------|
| 领域 | 物理/工程/几何 | 优化/博弈/决策 | 数据/统计/ML |
| 核心能力 | 物理建模、数学推导 | 算法设计、策略对比 | 特征工程、模型选择 |
| 数学工具 | PDE/ODE、几何建模 | 整数规划、图论、博弈论 | 回归、分类、聚类 |
| 推荐编程语言 | MATLAB | MATLAB / C++ | Python |
| 评审最看重 | 公式推导的物理依据 | 策略的完备性与对比 | 特征工程的合理性与模型解释 |
| 常见误区 | 只列公式不解释物理意义 | 只给结果不对比基线 | 堆砌 ML 模型不做特征分析 |

---

## A 题策略：物理/工程建模

### 常见题目类型
- 热传导/热力学（2018 高温作业服、2020 回焊炉）
- 力学/运动学（2022 波浪能装置、2024 板凳龙）
- 光学/几何建模（2021 FAST 反射面、2023 定日镜场）
- 流体/压力系统（2019 高压油管）

### 标准建模流程
1. **物理机制分析**：基于第一性原理，明确守恒律/本构关系
2. **合理简化假设**：对称性、小量忽略、准静态近似等
3. **数学建模**：建立 PDE/ODE 或几何方程，从假设出发逐级推导
4. **数值求解**：有限差分、有限元、数值积分等
5. **参数拟合**：利用实测数据反推模型参数，报告拟合优度（R²）
6. **结果验证**：将模型预测与实验/实测数据对比

### 推荐工具
- **MATLAB**（首选）：PDE Toolbox、ode45/ode15s、lsqcurvefit、CFD 初步
- Python（备选）：scipy.integrate、FiPy、FEniCS

### 图表类型偏好
- 物理量随空间/时间的变化曲线（最重要）
- 模型预测 vs 实验数据对比图（必备，用于验证）
- 系统结构/几何示意图（坐标轴、关键参数标注）
- 等值线图/云图（二维场分布）
- 参数灵敏度柱状图

### A 题易犯错误
1. **抄教科书公式而不做问题适配**：公式中的符号应对应题目的具体变量
2. **缺少物理机制解释**：每个方程前应有 1-2 句物理依据
3. **假设与后文脱节**：写了"忽略热辐射"但后文没验证忽略的合理性
4. **只给最终数值不给中间推导**：关键步骤的公式推导应展示
5. **参数取值依据不明**：每个参数值应说明来源（实测拟合/文献/经验公式）

### 获奖论文特征
- 数学推导密度高（公式占比 30%+ 的正文篇幅）
- 每个子问题至少一个"模型-实验对比图"
- 代码以 MATLAB 为主，附录含完整求解脚本
- 常引用工程热物理/力学领域教材或手册

## B 题策略：优化/博弈/决策

### 常见题目类型
- 调度优化（2018 RGV 动态调度）
- 策略博弈（2020 穿越沙漠、2019 同心鼓）
- 资源分配（2021 原材料订购与运输）
- 路径与定位（2022 无人机定位、2023 多波束测线）
- 生产决策（2024 生产决策优化）

### 标准建模流程
1. **明确目标与约束**：目标函数 + 决策变量 + 约束条件（数学化）
2. **模型选择**：根据问题规模、线性/非线性、连续/离散选择
3. **算法设计**：精确算法（LP/IP/DP）或启发式（GA/PSO/SA/ACO）
4. **多策略对比**：至少 2-3 种策略/算法对比，讨论优劣
5. **收敛性验证**：展示算法收敛曲线
6. **鲁棒性测试**：在不同工况/参数下验证策略稳定性

### 常用模型与算法
- **精确求解**：线性规划(LP)、整数规划(IP)、0-1 规划、动态规划(DP)
- **启发式算法**：遗传算法(GA)、粒子群(PSO)、模拟退火(SA)、蚁群(ACO)
- **博弈模型**：静态博弈、Stackelberg 博弈、演化博弈
- **图论**：最短路径、最大流、二分图匹配

### 推荐工具
- **MATLAB**：Optimization Toolbox、Global Optimization Toolbox
- **C++**：大规模搜索/模拟场景下性能优先
- Python：scipy.optimize、DEAP (GA)、pyswarms

### 图表类型偏好
- 算法流程图（必备）
- 多策略对比表/柱状图（核心，证明你的策略最优）
- 收敛曲线（迭代次数 vs 目标函数值）
- 调度甘特图/时间线图
- 灵敏度龙卷风图

### B 题易犯错误
1. **只给一种策略不做对比**：没有baseline，评委无法判断优劣
2. **缺乏最优性证明**：启发式算法需要讨论"为什么认为接近全局最优"
3. **约束条件遗漏**：解题后发现最优解违反题目约束
4. **代码结果与论文数据不一致**：算法输出的数字必须和论文完全一致
5. **过度依赖现成工具箱**：应展示对算法原理的理解，不要只调用一个 black-box 函数

### 获奖论文特征
- 算法设计部分包含流程图 + 伪代码
- 策略对比表（含 3+ 种策略的结果和效率对比）
- 蒙特卡洛模拟验证（1000+ 次实验报告置信区间）
- 代码以 MATLAB 和 C++ 为主（性能考虑）

## C 题策略：数据/统计/机器学习

### 常见题目类型
- 分类与鉴别（2022 玻璃文物分类）
- 预测与回归（2021 乙醇制备 C4 烯烃产率预测）
- 聚类与分级（2020 信贷风险评估、2023 蔬菜定价）
- 综合评价与决策（2024 农作物种植策略）

### 标准建模流程
1. **探索性数据分析 (EDA)**：描述性统计、分布检查、相关性分析
2. **数据预处理**：缺失值填补、异常值处理、标准化/归一化
3. **特征工程**：特征构造、特征选择（相关性/重要性/逐步回归）
4. **模型建立**：从简单到复杂（先做线性再做非线性）
5. **模型对比**：至少 3 种方法对比（如 线性回归 vs 随机森林 vs XGBoost）
6. **性能评估**：交叉验证、AUC/准确率/R²/MAPE 等
7. **结果解释**：特征重要性、SHAP 值、偏依赖图

### 常用模型（按复杂度递增）
- **基础**：线性回归、逻辑回归、多元回归
- **统计**：方差分析(ANOVA)、主成分分析(PCA)、因子分析
- **ML 经典**：决策树、随机森林、SVM、KNN
- **ML 进阶**：XGBoost、LightGBM、神经网络(BP)
- **评价方法**：熵权法、TOPSIS、AHP、模糊综合评价

### 推荐工具
- **Python**（首选）：pandas, scikit-learn, xgboost, matplotlib/seaborn
- MATLAB：Statistics and Machine Learning Toolbox

### 图表类型偏好
- 相关性热力图 / 散点图矩阵（EDA 阶段）
- 特征重要性排序图（建模阶段，必备）
- ROC 曲线 / PR 曲线（分类模型）
- 模型预测 vs 真实值散点图（回归模型）
- 残差分布图 / Q-Q 图（模型诊断）
- SHAP 依赖图（加分项，解释模型）

### C 题易犯错误
1. **不写 EDA 直接建模**：评委无法判断你对数据的理解
2. **堆砌 ML 模型没有理由**：每个模型的选择必须有理由，不能"试试看哪个好"
3. **只给准确率不给可解释性**：C 题评委重视"为什么这个变量重要"
4. **特征工程太少**：直接拿原始数据跑模型是最低级做法
5. **过度调参却轻视特征**：好的特征比微调超参数重要得多
6. **混淆训练集和测试集**：数据泄露 → 模型看似优秀，实际无效

### 获奖论文特征（C 题）
- EDA 占 1-2 页，有描述性统计表 + 分布图
- 特征工程过程详细记录（构造了什么特征、为什么、效果如何）
- 3+ 种模型对比表（含各模型超参数和性能指标）
- Python 代码附录，含数据预处理和模型训练全流程
- 结果解释段落（不只说"准确率 XX%"，还要说"这说明……"）

## 快速选题决策

根据队伍优势选择题目：

| 队伍优势 | 推荐选题 | 理由 |
|---------|---------|------|
| 物理/工科背景强 | A 题 | 需要工程直觉和物理建模能力 |
| 算法/编程强 | B 题 | 需要设计高效算法和大量编程 |
| 统计/数据处理强 | C 题 | 需要特征工程和模型调优经验 |
| MATLAB 熟练 | A/B 题 | 数值计算和优化工具箱 |
| Python 熟练 | C 题 | ML 生态和数据处理 |

> **C 题适合首次参赛**：门槛相对较低，有数据就能出结果。A/B 题适合冲高奖，但建模失败风险较大。
