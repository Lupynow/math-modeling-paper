# 图表规范与代码附录要求

基于 2018-2024 获奖论文的图表质量分析和最新比赛规则。

---

## 一、流程图

### 何时需要流程图

- **整体建模框架图**（问题分析章末）：展示所有子问题对应的模型及数据流，约 70% 获奖论文包含此图
- **算法流程图**（求解章节）：展示迭代/搜索过程，尤其重要于 B 题和包含复杂算法的 A 题

### 绘制规范

1. **框类型**：
   - 圆角矩形 → 输入/输出
   - 矩形 → 处理步骤/模型
   - 菱形 → 判断/分支
   - 圆柱形 → 数据/数据库

2. **箭头**：标注数据流向或条件（如"是/否"）

3. **配色**：
   - 使用 2-4 种颜色区分不同层次（输入层→处理层→输出层）
   - 避免高饱和度颜色（大红大绿），偏向学术配色（蓝/灰/橙）
   - 子问题用同色系不同深浅区分

4. **工具推荐**：
   - **draw.io / diagrams.net**（免费，导出 SVG/PNG）
   - **Visio**（如有正版）
   - **MATLAB**：可用 `digraph` + `plot` 绘制有向图
   - **Python**：`graphviz` 或 `networkx`

5. **标注**：中英文均可，国赛用中文，美赛用英文。字号统一，避免过大过小。

6. **格式**：导出为矢量图（PDF/SVG）插入 Word/LaTeX，避免直接截图

### 流程图示例结构

```
[数据输入] → [问题一：XX模型] → [结果1] 
                  ↓
            [问题二：YY模型] → [结果2]
                  ↓
            [问题三：ZZ模型] → [结果3]
                  ↓
            [模型检验与评价]
```

## 二、数据图表

### 通用规范

1. **标题位置**：表标题在上方，图标题在下方
2. **标题内容**：有描述性，不是"图1：结果图"，而是"图1：不同炉温曲线下的焊接质量对比"
3. **编号**：全文统一编号（图1、图2...），按出现顺序
4. **单位**：所有数值必须有单位（在坐标轴标签或表头中）
5. **字号**：图表内文字不小于正文的 80%

### 表格规范

- 使用**三线表**（顶线、栏目线、底线），中间不加竖线
- 数据对齐：数字右对齐，文字左对齐
- 有统计显著性时标注 `*` `**` `***`（附注在表下）

### 数据图规范

1. **坐标轴**：
   - 标注变量名 + 单位，如 `温度 (°C)` / `Temperature (°C)`
   - 坐标轴范围合适（不要太大留白，也不要太紧使数据贴边）
   - 刻度标签可读

2. **配色建议**：
   - 对比色：蓝 vs 橙，红 vs 蓝（避免红绿对比，红绿色盲不可分辨）
   - 同组数据用同色系（不同深浅）
   - 论文打印时可能是黑白的：确保灰度下也能区分线条（用不同线型：实线/虚线/点线）

3. **图例**：
   - 放在图内空白处或图下方
   - 线条数 > 4 条时考虑分图

4. **多面板图**：用 (a)(b)(c) 标注子图，统一主标题

### 核心图表类型

| 图表类型 | 适用场景 | 注意事项 |
|---------|---------|---------|
| 折线图 | 时间序列、参数扫描 | ≤4 条线，超过用分图 |
| 散点图 | 模型预测 vs 真实值、相关性 | 可加趋势线 + R² |
| 柱状图 | 分类对比、特征重要性排序 | 排序后展示，水平或垂直 |
| 热力图 | 相关性矩阵、二维分布 | 标注数值或颜色条 |
| 龙卷风图 | 灵敏度分析排序 | 按影响大小排序 |
| 箱线图 | 分布对比、异常值检测 | 标注中位数和 IQR |
| ROC 曲线 | 二分类模型评估 | 标注 AUC 值 |

### 工具推荐

- **Python**：matplotlib + seaborn（首选，输出 PDF 矢量图）
- **MATLAB**：原生 plot 函数（导出为 .eps 或 .pdf）
- **R**：ggplot2（最专业的统计图，美赛推荐）

## 三、算法伪代码

### 何时需要

- B 题（优化/博弈）：强烈推荐，展示算法设计思路
- A 题（数值求解）：如有迭代算法，推荐
- C 题（ML）：不必须，但如果自行实现了算法（非调包），推荐

### 格式规范

```
Algorithm 1: 自适应遗传算法求解XX问题

Input: 种群大小 N, 最大迭代次数 Tmax, 交叉概率 pc, 变异概率 pm
Output: 最优解 x* 及其目标函数值 f*

1  Initialize population P(0) with N individuals
2  Evaluate fitness for each individual in P(0)
3  for t = 1 to Tmax do
4      Select parents from P(t-1) by tournament selection
5      Apply adaptive crossover (pc ∝ 1/fitness)
6      Apply adaptive mutation (pm ∝ fitness diversity)
7      Evaluate fitness of offspring
8      Merge and select N elite individuals → P(t)
9      if convergence criterion met then
10         break
11  return Best individual x* and its fitness f*
```

### 规范要点

- 用英文写（国赛论文中也可用英文伪代码）
- 标注 Input / Output
- 行号对齐
- 关键词用 **bold**
- 代码块在正文中引用："如 Algorithm 1 所示"
- 不要写具体语言语法（如 `for i in range(N)`），保持语言无关

## 四、代码附录

### 基本要求

2026 年国赛规则：**缺少必要源程序或程序不能运行，可能被取消评奖资格。**

### 代码组织

1. **分节注释**：每个子问题的代码前用明显注释分隔
   ```matlab
   %% ==========================================
   %% 问题一：XXX模型 - 数据预处理与模型求解
   %% ==========================================
   ```

2. **命名一致性**：代码中的变量名应与论文中使用的符号一致
   - 论文中写 `T_peak`，代码中也用 `T_peak`
   - 论文中写 `α`，代码中用 `alpha`
   - 论文中写 `λ_max`，代码中用 `lambda_max`

3. **关键参数注释**：对模型的超参数和取值依据加注释
   ```python
   learning_rate = 0.01  # 经网格搜索确定（见论文 5.2.3 节）
   ```

4. **输出一致性**：代码运行结果应与论文中报告的数据完全一致
   - 论文声称 RMSE=12.4，代码输出就应是 12.4
   - 建议在代码末尾加 `assert` 验证关键结果

### 按语言的选择与规范

#### MATLAB（A/B 题推荐）
```
优点：数值计算强，PDE/优化工具箱成熟
规范：
- 用 %% 分节（对应论文子问题）
- 函数文件独立命名（如 model_A_heat_transfer.m）
- 主脚本命名为 main.m，放在附录最前
- 用 saveas / print 导出图片时注释用途
```

#### Python（C 题推荐）
```
优点：ML 生态，数据处理方便
规范：
- 用 # %% 或清晰注释分节
- 统一 import 放在文件开头
- 每个模型独立函数（如 def model_xgboost():）
- 关键步骤输出用 print() 且注释含义
- requirements.txt 或注释说明依赖包版本
```

#### C++（B 题大规模计算时推荐）
```
优点：性能高，适合大规模搜索/模拟
规范：
- 每个策略/算法独立函数
- 关键参数用 const 声明并注释来源
- 输出格式清晰，方便直接复制到论文
```

### 附录中不需要的

- 完整的第三方库文件（只写 import / #include）
- 调试用的打印语句
- 未使用的函数或变量
- 被注释掉的旧版本代码
- 数据文件本身（只放处理逻辑，不放数百行数据）

### 附录检查清单

- [ ] 代码可以完整运行（不依赖未提供的文件）
- [ ] 每个子问题有明确的分节标注
- [ ] 论文中声称的方法在代码中实现（不是线性回归论文 + XGBoost 代码）
- [ ] 关键输出值与论文完全一致
- [ ] 变量名与论文符号一致
- [ ] 代码有基本注释（每个函数/主要步骤）
- [ ] 如果是 Python，注释了依赖包版本
